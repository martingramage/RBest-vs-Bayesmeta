1 Introduction

In this vignette we explain the workflow of two different packages used to derive meta‑analytic predictive (MAP) prior distributions, which facilitate the incorporation of historical information into clinical trials. In both scenarios, a Bayesian hierarchical model is built to link the endpoint to latent parameters, which themselves are treated as parameters with an additional layer of hyperparameters. This structure allows us to use the data to update the initial prior distributions of the latent parameters, obtain their posterior distributions, and effectively summarize the information from past studies into informative priors. However, since the posterior distribution is not usually analytically tractable, different approaches are required to approximate it.

1.1 RBesT:

The R Bayesian evidence synthesis Tools (RBesT) is based on MCMC computation to obtain an empirical assimptotically correct posterior distribution, which is later fitted to a parametric distribution.

1.2 Bayesmeta:

Bayesmeta uses analytic numerical integration under normality assumptions, which makes the computation faster but requires approximations when dealing with non‑normal endpoints. It produces a fully analytical mixture distribution that can be directly used in the subsequent steps of the workflow.

3 Prior Derivation

3.1 RBesT

3.1.1 Meta-Analytic-Predictive Analysis

The gMAP function performs the meta-analysis and the prediction, which yields the MAP prior. The analysis is run using stochastic Markov-Chain-Monte-Carlo with Stan. In order to make results exactly reproducible, the set.seed function must be called prior to calling gMAP .

A key parameter in a meta-analysis is the between-trial heterogeneity parameter \(\tau\) which controls the amount of borrowing from historical information for the estimation of the population mean will occur. As we often have only few historical trials, the prior is important. For binary endpoints with an expected response rate of 20%-80% we recommend a conservative HalfNormal(0,1) prior as a default. Please refer to the help-page of gMAP for more information.

3.1.2 Specifying the prior distributions

As explained in the RBesT reference for Generalized Linear Models, the prior distribution for the regression coefficients \(\beta\) is defined as a normal, being recommended to specify a beta.prior. As the data from the studies is sufficiently informative, we use a vague prior where a mean of 0 is set per default. Additionally the standard deviation is set to 2 for the binary case Schmidli et al. 2014.

On the other hand, due to the limited number of historical trials, it is often challenging to reliably estimate the distribution of the heterogeneity parameter \(\tau\) from the data alone. Therefore, the prior distribution for \(\tau\) must be explicitly specified, selected from one of the following supported options: "HalfNormal", "TruncNormal", "Uniform", "Gamma", "InvGamma", "LogNormal", "TruncCauchy", "Exp", or "Fixed".

A uniform prior can only yield an integrable proper posterior with at least 3 studies, and it is commonly considered overly conservative. Therefore, in order to choose a proper prior for the heterogeneity, the distribution not only needs to be flexible near zero (avoiding favoring small values on the \(\tau\) scale), but also to decay properly Röver 2021.

Although Sebastian Weber has commented on his preference to use a Penalized Complexity prior, he also stresses that the accepted prior in literature is a Half-Normal Weber 2024, which maintains a conservative approach and should be used if no proven reasons dismiss it for the case study. It ensures a flat behavior close to 0, and has a Gaussian-like decay.

As for the scale parameter (s), it is related to \(\sigma\) and it is endpoint specific. For binary endpoints, a Half-Normal prior with a standard deviation of 1 puts approximately 5% probability for values of \(\tau\) greater than 2. While value of \(\tau\) = 2 corresponds to very large between-trial variability on the log-odds-scale, and would result in minimal borrowing from historical data Schmidli et al. 2014. Among literature agreement, a scale parameter of 0.5 is often preferred, as it reflects the empirical evidence that \(\tau\) usually falls on moderate values Turner et al. 2012Turner et al. 2015Günhan et al. 2020. It assigns approximately half of the probability to values that range from 0.2 to 0.5, which ensures that most of the prior mass lies where the real heterogeneity has often been observed. Additionally, in cases where we expect a higher homogeneity among studies (e.g, Phase III twin clinical trials), we can reduce the standard deviation to better reflect our initial information.

3.1.4 Underlying Stan computation

When performing MCMC-based Bayesian inference in RBesT, it is important to remember that the computation is powered by Stan—a robust probabilistic programming engine known for Hamiltonian Monte Carlo (HMC) sampling. Consequently, common issues such as poor convergence or low effective sample size of the simulated MCMC sample (not to confuse with the effective sample size of the MAP distribution) can be addressed using standard solutions developed for Stan workflows. One straightforward remedy is to augment the number of iterations, allowing the sampler more opportunity to explore the posterior space. Additional transferable strategies include:

  • Increasing adapt_delta to improve step acceptance and reduce divergences
  • Raising max_treedepth to enable deeper exploration of complex posterior geometries
  • Running multiple chains and monitoring \(\hat{R}\) to assess convergence
  • Applying sufficient warm-up (burn-in) iterations
  • Inspecting trace plots for mixing and autocorrelation

3.1.5 Parametric Approximation

As a next step, the MAP prior, represented numerically using a large MCMC simulation sample, is converted to a parametric representation with the automixfit function. This function fits a parametric mixture representation using expectation-maximization (EM). The number of mixture components to best describe the MAP is chosen automatically. By default the function chooses from up to 4 elements using the Akaike information criterion (AIC), whose penalty parameter is set in 6. Again, the plot function produces a graphical diagnostic which allows the user to assess whether the marginal mixture density (shown in black) matches well with the histogram of the MAP MCMC sample.

While a kernel-density estimate from the MCMC sample or a Rao-Blackwellized density estimate can characterize the MAP prior, these approaches often involve a large number of parameters, which complicates both their interpretation and application in the context of a new trial. A more practical and parsimonious alternative is to approximate the prior using mixtures of conjugate distributions. In the binary response, this corresponds to an approximation of a mixture of beta distributions Schmidli et al. 2014.

3.2 Bayesmeta

In order to enrich our analysis we can compare the results with the ones we would obtain using the Bayesmeta package approach. The bayesmeta function only supports NNHM (Normal-Normal Hierarchical Models) with known \(\sigma\), so we need to adapt the meta-analysis to our case.

As referred in their instructions Röver 2020, we first need to transform our data with the logit link using the function escalc, in order to approximate the binary endpoint to the mean and standard deviation of the effect size for each study. We then use the function bayesmeta to obtain a bayesmeta class object. This object allows, among other options, to directly compute the posterior predictive probability density for \(\theta\), which we expect to be a mix of normal distributions.

Its underlying method collapses the hierarchical model to integrate the latent parameters and express the endpoint directly in relation to the hyperparameters.

First, we need to remember that we have changed the endpoint from binomial to normal. Given \(k\) independent studies with estimates of the effect \(y_i\) with known means \(\theta_i\) and standard errors \(\sigma_i\), the Bayesian random-effects meta-analysis model assumes the following:

  1. Sampling model (study-level observations):

\[ y_i \mid \theta_i, \sigma_i \sim \mathcal{N}(\theta_i, \sigma_i^2)\text{.} \]

  1. Random effects model (study-level true effects):

\[ \theta_i \mid \mu, \tau \sim \mathcal{N}(\mu, \tau^2)\text{.} \]

  1. Marginal model (after integrating out \(\theta_i\)):

\[ y_i \mid \mu, \tau, \sigma_i \sim \mathcal{N}(\mu, \sigma_i^2 + \tau^2)\text{,} \]

where:

- \(\mu\) is the overall effect size.

- \(\tau^2\) captures between-study heterogeneity.

- \(\sigma_i^2\) is the known sampling variance for study \(i\).

Using the improper uniform prior for the global effect \(\mu\) (i.e., \(p(\mu) \propto 1\)), we can derive the marginal likelihood, marginalized over \(\mu\):

\[ p(\mathbf{y} \mid \tau, \boldsymbol{\sigma}) = \int p(\mathbf{y} \mid \mu, \tau, \boldsymbol{\sigma}) \cdot p(\mu) \, d\mu\text{.} \]

As long as a uniform or normal prior for the global effect \(\mu\) is used, the global effect’s conditional posterior distribution for a given heterogeneity value, \(p(\mu|\tau,y,\sigma)\), is normal with mean \(\hat\mu(\tau)\) and standard deviation \(\hat\sigma(\tau)\).

Where \(\hat{\mu}(\tau)\) is the conditional posterior mean of \(\mu\) given heterogeneity \(\tau\), defined by:

\[ \hat{\mu}(\tau) = \mathbb{E}[\mu \mid \tau, \mathbf{y}, \boldsymbol{\sigma}] = \frac{\sum_{i=1}^{k} \frac{y_i}{\sigma_i^2 + \tau^2}}{\sum_{i=1}^{k} \frac{1}{\sigma_i^2 + \tau^2}}\text{,} \]

and the corresponding standard deviation is:

\[ \hat{\sigma}(\tau) = \sqrt{ \mathrm{Var}(\mu \mid \tau, \mathbf{y}, \boldsymbol{\sigma}) } = \sqrt{ \frac{1}{\sum_{i=1}^{k} \frac{1}{\sigma_i^2 + \tau^2}} } \text{.} \]

Therefore, for the normal-normal hierarchical model (NNHM), this integral evaluates to:

\[ p(\mathbf{y} \mid \tau, \boldsymbol{\sigma}) = (2\pi)^{-k/2} \cdot \left( \prod_{i=1}^{k} \frac{1}{\sqrt{\sigma_i^2 + \tau^2}} \right) \cdot \exp\left( -\frac{1}{2} \sum_{i=1}^{k} \frac{(y_i - \hat{\mu}(\tau))^2}{\sigma_i^2 + \tau^2} \right) \cdot \frac{1}{\sqrt{\sum_{i=1}^{k} \frac{1}{\sigma_i^2 + \tau^2}}}\text{.} \]

Having derived the marginal likelihood \(p(\mathbf{y} \mid \tau, \boldsymbol{\sigma})\), the (one-dimensional) marginal posterior density of \(\tau\) may be computed (up to a normalizing constant) by multiplication with the heterogeneity prior:

\[ p(\tau \mid \mathbf{y}, \boldsymbol{\sigma}) \propto p(\mathbf{y} \mid \tau, \boldsymbol{\sigma}) \cdot p(\tau)\text{.} \]

Finally, the marginal distribution of the global effect \(\mu\), which forms a normal mixture, can be approximated by using a discrete grid of \(\tau\) values and summing the conditional normal distributions, weighted by the marginal density of \(\tau\):

\[ p(\mu\mid \mathbf{y}, \boldsymbol{\sigma}) = \int p(\mu \mid \tau) \cdot p(\tau) \, d\tau \approx \sum_j p(\mu \mid \tau_j) \cdot w_j\text{,} \]where:

- \(p(\mu \mid \tau_j)\) is the conditional posterior distribution of \(\mu\) given \(\tau_j \text{.}\)

- \(w_j\) is the posterior weight assigned to grid point \(\tau_j\), proportional to \(p(\mathbf{y} \mid \tau_j) \cdot p(\tau_j)\text{.}\)

After both marginal posterior distributions for the global effect (\(\mu\)) and the heterogeneity across studies (\(\tau\)) have been obtained, bayesmeta can directly derive the marginal posterior predictive distribution for the effect (\(\theta\)) by specifying the argument predict=TRUE. Choosing among $dposterior(), $pposterior(), $qposterior(), $rposterior(), we can access the probability density, the cumulative distribution, the quantile function, or generate a random sample of the posterior predictive distribution for \(\theta\), respectively.

In order to later compare the results to those obtained in the RBesT workflow, we need to transform the mixed normal distribution of the effect to the original response scale. The resulting distribution is a mixed logit-normal distribution, as it is composed of variables that, when logit-transformed, follow normal distributions. The pushforward measure for this transformation can be expressed analytically using a change of variables, which must include the appropriate Jacobian correction Frederic et al. 2003Betancourt 2023:

If a random variable \(\theta \sim \mathcal{N}(\mu, \sigma^2)\), and you define:

\[ p = \text{logit}^{-1}(\theta) = \frac{1}{1 + e^{-\theta}}, \]

then the transformed variable \(p\) follows a logit-normal distribution, and its density is given by:

\[ f(p; \mu, \sigma) = \frac{1}{\sigma \sqrt{2\pi} \cdot p(1 - p)} \cdot \exp\left( -\frac{(\text{logit}(p) - \mu)^2}{2\sigma^2} \right), \]

where \(\text{logit}(p) = \log\left( \frac{p}{1 - p} \right)\).

This expression is valid for \(p \in (0, 1)\), and describes the probability density function of a logit-normal distribution derived from a normal distribution on the log-odds scale.

Additionally, pushforward measures conserve linearity Betancourt 2023Schilling 2017, so the transformation of each component of the mixed normal distribution commutes with the transformation of the entire mixed distribution. Note that independence of components is not required, as the preservation of linearity in image measures arises from the structure of the linear operator acting on the measure space, and not from the independence of random variables. Therefore, we can obtain the mixed logit-normal distribution by transforming the weighted components and mixing them, regardless of whether the components are independent.

Code 
#Logit-normal trasformation
dlogitnorm <- function(p, mu, sigma) {
  theta <- log(p / (1 - p))
  (1 / (sigma * sqrt(2 * pi))) * (1 / (p * (1 - p))) * exp(- (theta - mu)^2 / (2 * sigma^2))
}

#Extract mixture components
weights <- map_bm$support[,"weight"]
mus     <- map_bm$support[,"mean"]

dlogitnorm_mixture <- function(p, weights, mus, sigmas) {
  rowSums(sapply(1:length(weights), function(j) {
    weights[j] * dlogitnorm(p, mu = mus[j], sigma = sigmas[j])
  }))
}

logitnorm_moment_k <- function(mu, sigma, k) {
  integrate(function(p) {
    theta <- log(p / (1 - p))

5 ESS

The (usual) intended use of a (MAP) prior is to reduce the number of control patients in the trial. The prior can be considered equivalent to a number of experimental observations, which is called the effective sample size (ESS) of the prior.

A summary of this explanation based on Neuenschwander et al. 2020 for the estimation of the ESS is presented bellow:

5.1 Moments method

It is based on computing the mean and standard deviation from the parameterized mixture distribution (estimated using EM if it was obtained through simulation). Next, a conjugate distribution that matches these moments allows for easier estimation for the ESS. Which can be derived from the updating rule:

  • Beta Prior (for Binomial or Bernoulli data):
    \[\text{ESS} = a + b\]

  • Gamma Prior (for Poisson or Exponential data):
    \[\text{ESS} = b\]

  • Normal Prior (for Normal likelihood with known variance):
    If the prior is \(\mathcal{N}(\mu, \frac{\sigma^2}{n_0})\), then:
    \[\text{ESS} = n_0\]

Or can be obtained from the variance and precision ratio methods.

5.1.1 Variance & precision ratio methods

It is based on comparing the variance (or precision) of the prior to the variance of an estimator \(Y_N\) for the parameter from a sample of size \(N\), averaged over the prior distribution of the parameter. ESS is the \(N\) for which the two variances are the same.

\[ \text{ESS} = \frac{N \cdot \mathbb{E}_\theta[\text{Var}(Y_N \mid \theta)]}{\text{Var}(\theta)}\text{.} \]

Using \(i_F(\theta)\) and \(i_F(Y;\theta)\) as the expected and observed Fisher information for one information unit respectively, with:

\[ i_F(\theta) = -\mathbb{E}_{Y_1 \mid \theta} \left[ \frac{d^2}{d\theta^2} \log f(Y_1 \mid \theta) \right]\text{,} \]

we obtain the ESS for the :

\[ \text{ESS}_{\text{VR}} = \frac{\mathbb{E}_\theta[i_F^{-1}(\theta)]}{\text{Var}(\theta)}\text{,} \]

and

\[ \text{ESS}_{\text{PR}} = \frac{1}{\text{Var}(\theta)} \cdot \mathbb{E}_\theta[i_F(\theta)]\text{.} \]

5.2 Morita method

It defines the ESS as the number of observations (\(m\)) that would make the posterior information from a vague prior with the same mean match the information content of the actual prior.

The ESS is given by:

\[ \text{ESS}_{\text{MTM}} = \frac{i(p(\overline{\theta})) - i(p_0(\overline{\theta}))}{\mathbb{E}_{Y_1}[i_F(Y_1; \theta)]}\text{.} \]

Where

  • \(i(p(\overline{\theta})) = -\frac{d^2}{d\theta^2} \log p(\theta) \big|_{\theta = \overline{\theta}}\text{ .}\)

    → Prior information (curvature of the actual prior).

  • \(i(p_0(\overline{\theta})) = -\frac{d^2}{d\theta^2} \log p_0(\theta) \big|_{\theta = \overline{\theta}}\text{ .}\)

    → Information of a vague prior with the same mean as \(p(\theta)\).

  • \(i_F(Y_1; \theta)\text{.}\)
    → Fisher information from one observation.

  • \(\mathbb{E}_{Y_1}[\cdot]\text{.}\)
    → Expectation over the prior predictive distribution.

5.3 ELIR method

Instead of evaluating the information ratio at the prior mean or mode, ELIR averages the ratio of prior curvature to Fisher information across all plausible values of the parameter (in this case designed by \(\theta\)).

\[ \text{ESS}_{\text{ELIR}} = \mathbb{E}_\theta \left[ \frac{i(p(\theta))}{i_F(\theta)} \right]\text{.} \]

Where

  • \(i(p(\theta)) = -\frac{d^2}{d\theta^2} \log p(\theta)\text{.}\)
    → Prior information (curvature of the log-prior).

  • \(i_F(\theta) = \mathbb{E}_{Y \mid \theta} \left[ -\frac{d^2}{d\theta^2} \log f(Y \mid \theta) \right]\text{.}\)
    → Fisher information from one observation.

  • The expectation \(\mathbb{E}_\theta[\cdot]\) is taken over the prior distribution.

5.4 Choosing for the method

As previously noted, the lack of convergence in effective sample size (ESS) estimates across different methods necessitates a clear choice of approach. The ELIR method emerges as the only predictively consistent option, making it the preferred technique for accurately quantifying information in terms of an equivalent number of observations. However, its applicability is currently limited to one-dimensional parameters, and further research is needed to address the extension of its benefits to higher-dimensional settings. Additionally, in situations where data-prior conflict is present, the advantages of the ELIR method diminish, given the reduced influence of the prior in the calculation of ESS.

5.4.1 Beta mixture (RBesT)

The ESS can be calculated in RBesT with the ess function. The default method in RBesT is the elir approach Neuenschwander et al. 2020 which results in reasonable ESS estimates. The moment matching approach leads to conservative (small) ESS estimates while the Morita Morita et al. 2008 method tends to estimates liberal (large) ESS estimates when used with mixtures:

# Introduction

In this vignette we explain the workflow of two different packages used to derive meta‑analytic predictive (MAP) prior distributions, which facilitate the incorporation of historical information into clinical trials. In both scenarios, a Bayesian hierarchical model is built to link the endpoint to latent parameters, which themselves are treated as parameters with an additional layer of hyperparameters. This structure allows us to use the data to update the initial prior distributions of the latent parameters, obtain their posterior distributions, and effectively summarize the information from past studies into informative priors. However, since the posterior distribution is not usually analytically tractable, different approaches are required to approximate it.

## RBesT:

The R Bayesian evidence synthesis Tools (RBesT) is based on MCMC computation to obtain an empirical assimptotically correct posterior distribution, which is later fitted to a parametric distribution.

## Bayesmeta:

Bayesmeta uses analytic numerical integration under normality assumptions, which makes the computation faster but requires approximations when dealing with non‑normal endpoints. It produces a fully analytical mixture distribution that can be directly used in the subsequent steps of the workflow.

# Prior Derivation

## RBesT

### Meta-Analytic-Predictive Analysis

The **`gMAP`** function performs the meta-analysis and the prediction, which yields the MAP prior. The analysis is run using stochastic Markov-Chain-Monte-Carlo with Stan. In order to make results exactly reproducible, the `set.seed` function must be called prior to calling **`gMAP`** .

A key parameter in a meta-analysis is the between-trial heterogeneity parameter $\tau$ which controls the amount of borrowing from historical information for the estimation of the population mean will occur. As we often have only few historical trials, the prior is important. For binary endpoints with an expected response rate of 20%-80% we recommend a conservative `HalfNormal(0,1)` prior as a default. Please refer to the help-page of **`gMAP`** for more information.

### Specifying the prior distributions

As explained in the [RBesT reference for Generalized Linear Models](https://opensource.nibr.com/RBesT/reference/gMAP.html#prior-specification), the prior distribution for the regression coefficients $\beta$ is defined as a normal, being recommended to specify a `beta.prior`. As the data from the studies is sufficiently informative, we use a vague prior where a mean of 0 is set per default. Additionally the standard deviation is set to 2 for the binary case [@schmidli2014robust].

On the other hand, due to the limited number of historical trials, it is often challenging to reliably estimate the distribution of the heterogeneity parameter $\tau$ from the data alone. Therefore, the prior distribution for $\tau$ must be explicitly specified, selected from one of the following supported options: `"HalfNormal"`, `"TruncNormal"`, `"Uniform"`, `"Gamma"`, `"InvGamma"`, `"LogNormal"`, `"TruncCauchy"`, `"Exp"`, or `"Fixed"`.

A uniform prior can only yield an integrable proper posterior with at least 3 studies, and it is commonly considered overly conservative. Therefore, in order to choose a proper prior for the heterogeneity, the distribution not only needs to be flexible near zero (avoiding favoring small values on the $\tau$ scale), but also to decay properly [@röver2021predictively].

Although Sebastian Weber has commented on his preference to use a Penalized Complexity prior, he also stresses that the accepted prior in literature is a Half-Normal [@weber2024applied], which maintains a conservative approach and should be used if no proven reasons dismiss it for the case study. It ensures a flat behavior close to 0, and has a Gaussian-like decay.

As for the scale parameter (*s)*, it is related to $\sigma$ and it is endpoint specific. For binary endpoints, a Half-Normal prior with a standard deviation of 1 puts approximately 5% probability for values of $\tau$ greater than 2. While value of $\tau$ = 2 corresponds to very large between-trial variability on the log-odds-scale, and would result in minimal borrowing from historical data [@schmidli2014robust]. Among literature agreement, a scale parameter of 0.5 is often preferred, as it reflects the empirical evidence that $\tau$ usually falls on moderate values [@turner2012predicting; @turner2015predictive; @gunhan2020random]. It assigns approximately half of the probability to values that range from 0.2 to 0.5, which ensures that most of the prior mass lies where the real heterogeneity has often been observed. Additionally, in cases where we expect a higher homogeneity among studies (e.g, Phase III twin clinical trials), we can reduce the standard deviation to better reflect our initial information.

### Underlying Stan computation

When performing MCMC-based Bayesian inference in `RBesT`, it is important to remember that the computation is powered by Stan—a robust probabilistic programming engine known for Hamiltonian Monte Carlo (HMC) sampling. Consequently, common issues such as poor convergence or low effective sample size of the simulated MCMC sample (not to confuse with the effective sample size of the MAP distribution) can be addressed using standard solutions developed for [Stan workflows](https://mc-stan.org/rstan/articles/rstan.html#sampling-difficulties). One straightforward remedy is to augment the number of iterations, allowing the sampler more opportunity to explore the posterior space. Additional transferable strategies include:

-   Increasing `adapt_delta` to improve step acceptance and reduce divergences
-   Raising `max_treedepth` to enable deeper exploration of complex posterior geometries
-   Running multiple chains and monitoring $\hat{R}$ to assess convergence
-   Applying sufficient warm-up (burn-in) iterations
-   Inspecting trace plots for mixing and autocorrelation

### Parametric Approximation

As a next step, the MAP prior, represented numerically using a large MCMC simulation sample, is converted to a parametric representation with the **`automixfit`** function. This function fits a parametric mixture representation using expectation-maximization (EM). The number of mixture components to best describe the MAP is chosen automatically. By default the function chooses from up to 4 elements using the Akaike information criterion (AIC), whose penalty parameter is set in 6. Again, the `plot` function produces a graphical diagnostic which allows the user to assess whether the marginal mixture density (shown in black) matches well with the histogram of the MAP MCMC sample.

While a kernel-density estimate from the MCMC sample or a Rao-Blackwellized density estimate can characterize the MAP prior, these approaches often involve a large number of parameters, which complicates both their interpretation and application in the context of a new trial. A more practical and parsimonious alternative is to approximate the prior using mixtures of conjugate distributions. In the binary response, this corresponds to an approximation of a mixture of beta distributions [@schmidli2014robust].

## Bayesmeta

In order to enrich our analysis we can compare the results with the ones we would obtain using the `Bayesmeta` package approach. The `bayesmeta` function only supports NNHM (Normal-Normal Hierarchical Models) with known $\sigma$, so we need to adapt the meta-analysis to our case.

Its underlying method collapses the hierarchical model to integrate the latent parameters and express the endpoint directly in relation to the hyperparameters.

First, we need to remember that we have changed the endpoint from binomial to normal. Given $k$ independent studies with estimates of the effect $y_i$ with known means $\theta_i$ and standard errors $\sigma_i$, the Bayesian random-effects meta-analysis model assumes the following:

1.  Sampling model (study-level observations):

$$ y_i \mid \theta_i, \sigma_i \sim \mathcal{N}(\theta_i, \sigma_i^2)\text{.} $$

2.  Random effects model (study-level true effects):

$$ \theta_i \mid \mu, \tau \sim \mathcal{N}(\mu, \tau^2)\text{.} $$

3.  Marginal model (after integrating out $\theta_i$):

$$ y_i \mid \mu, \tau, \sigma_i \sim \mathcal{N}(\mu, \sigma_i^2 + \tau^2)\text{,} $$

where:

\- $\mu$ is the overall effect size.

\- $\tau^2$ captures between-study heterogeneity.

\- $\sigma_i^2$ is the known sampling variance for study $i$.

Using the improper uniform prior for the global effect $\mu$ (i.e., $p(\mu) \propto 1$), we can derive the marginal likelihood, marginalized over $\mu$:

$$ p(\mathbf{y} \mid \tau, \boldsymbol{\sigma}) = \int p(\mathbf{y} \mid \mu, \tau, \boldsymbol{\sigma}) \cdot p(\mu) \, d\mu\text{.} $$

As long as a uniform or normal prior for the global effect $\mu$ is used, the global effect’s conditional posterior distribution for a given heterogeneity value, $p(\mu|\tau,y,\sigma)$, is normal with mean $\hat\mu(\tau)$ and standard deviation $\hat\sigma(\tau)$.

Where $\hat{\mu}(\tau)$ is the conditional posterior mean of $\mu$ given heterogeneity $\tau$, defined by:

$$ \hat{\mu}(\tau) = \mathbb{E}[\mu \mid \tau, \mathbf{y}, \boldsymbol{\sigma}] = \frac{\sum_{i=1}^{k} \frac{y_i}{\sigma_i^2 + \tau^2}}{\sum_{i=1}^{k} \frac{1}{\sigma_i^2 + \tau^2}}\text{,} $$

and the corresponding standard deviation is:

$$ \hat{\sigma}(\tau) = \sqrt{ \mathrm{Var}(\mu \mid \tau, \mathbf{y}, \boldsymbol{\sigma}) } = \sqrt{ \frac{1}{\sum_{i=1}^{k} \frac{1}{\sigma_i^2 + \tau^2}} } \text{.} $$

Therefore, for the normal-normal hierarchical model (NNHM), this integral evaluates to:

$$ p(\mathbf{y} \mid \tau, \boldsymbol{\sigma}) =  (2\pi)^{-k/2} \cdot \left( \prod_{i=1}^{k} \frac{1}{\sqrt{\sigma_i^2 + \tau^2}} \right) \cdot \exp\left( -\frac{1}{2} \sum_{i=1}^{k} \frac{(y_i - \hat{\mu}(\tau))^2}{\sigma_i^2 + \tau^2} \right) \cdot \frac{1}{\sqrt{\sum_{i=1}^{k} \frac{1}{\sigma_i^2 + \tau^2}}}\text{.} $$

Having derived the marginal likelihood $p(\mathbf{y} \mid \tau, \boldsymbol{\sigma})$, the (one-dimensional) marginal posterior density of $\tau$ may be computed (up to a normalizing constant) by multiplication with the heterogeneity prior:

$$ p(\tau \mid \mathbf{y}, \boldsymbol{\sigma}) \propto p(\mathbf{y} \mid \tau, \boldsymbol{\sigma}) \cdot p(\tau)\text{.} $$

Finally, the marginal distribution of the global effect $\mu$, which forms a normal mixture, can be approximated by using a discrete grid of $\tau$ values and summing the conditional normal distributions, weighted by the marginal density of $\tau$:

$$ p(\mu\mid \mathbf{y}, \boldsymbol{\sigma}) = \int p(\mu \mid \tau) \cdot p(\tau) \, d\tau \approx \sum_j p(\mu \mid \tau_j) \cdot w_j\text{,} $$where:

\- $p(\mu \mid \tau_j)$ is the conditional posterior distribution of $\mu$ given $\tau_j \text{.}$

\- $w_j$ is the posterior weight assigned to grid point $\tau_j$, proportional to $p(\mathbf{y} \mid \tau_j) \cdot p(\tau_j)\text{.}$

After both marginal posterior distributions for the global effect ($\mu$) and the heterogeneity across studies ($\tau$) have been obtained, `bayesmeta` can directly derive the marginal posterior predictive distribution for the effect ($\theta$) by specifying the argument `predict=TRUE`. Choosing among `$dposterior()`, `$pposterior()`, `$qposterior()`, `$rposterior()`, we can access the probability density, the cumulative distribution, the quantile function, or generate a random sample of the posterior predictive distribution for $\theta$, respectively.

In order to later compare the results to those obtained in the RBesT workflow, we need to transform the mixed normal distribution of the effect to the original response scale. The resulting distribution is a mixed logit-normal distribution, as it is composed of variables that, when logit-transformed, follow normal distributions. The pushforward measure for this transformation can be expressed analytically using a change of variables, which must include the appropriate Jacobian correction [@frederic2003logit; @betancourt2023transforming]:

If a random variable $\theta \sim \mathcal{N}(\mu, \sigma^2)$, and you define:

$$ p = \text{logit}^{-1}(\theta) = \frac{1}{1 + e^{-\theta}}, $$

then the transformed variable $p$ follows a logit-normal distribution, and its density is given by:

$$ f(p; \mu, \sigma) = \frac{1}{\sigma \sqrt{2\pi} \cdot p(1 - p)} \cdot \exp\left( -\frac{(\text{logit}(p) - \mu)^2}{2\sigma^2} \right), $$

where $\text{logit}(p) = \log\left( \frac{p}{1 - p} \right)$.

This expression is valid for $p \in (0, 1)$, and describes the probability density function of a logit-normal distribution derived from a normal distribution on the log-odds scale.

Additionally, pushforward measures conserve linearity [@betancourt2023transforming; @schilling2017measures], so the transformation of each component of the mixed normal distribution commutes with the transformation of the entire mixed distribution. Note that independence of components is not required, as the preservation of linearity in image measures arises from the structure of the linear operator acting on the measure space, and not from the independence of random variables. Therefore, we can obtain the mixed logit-normal distribution by transforming the weighted components and mixing them, regardless of whether the components are independent.

```{r, message=FALSE, warning=FALSE}
#Logit-normal trasformation
dlogitnorm <- function(p, mu, sigma) {
  theta <- log(p / (1 - p))
  (1 / (sigma * sqrt(2 * pi))) * (1 / (p * (1 - p))) * exp(- (theta - mu)^2 / (2 * sigma^2))
}

  }))
}
```

-   Moments

To obtain the raw moments of a mixture distribution, we can extend the linearity of expectation to pushforward measures and compute the weighted sum of the moments of each transformed component [@casella2024statistical]. For univariate distributions, central moments can be derived from raw moments without explicitly accounting for covariance between components [@papoulis2002probability]. However, the logit-normal distribution lacks closed-form expressions for its moments, so they must be computed numerically. In order to do so we can use the function `momentsLogitnorm()` from the package `logitnorm()`, which calculates them using adaptative quadrature [@frederic2003logit].

```{r, message=FALSE, warning=FALSE}

logitnorm_moment_k <- function(mu, sigma, k) {
  integrate(function(p) {
    theta <- log(p / (1 - p))
    dens <- (1 / (sigma * sqrt(2 * pi))) * (1 / (p * (1 - p))) *
            exp(- (theta - mu)^2 / (2 * sigma^2))
    p^k * dens
  }, lower = 0, upper = 1, rel.tol = 1e-10)$value
}

#Moments for the mixture
moment_k_mixture <- function(k, weights, mus, sigmas) {
  sum(sapply(1:length(weights), function(j) {
    weights[j] * logitnorm_moment_k(mu = mus[j], sigma = sigmas[j], k = k)
  }))
}
```

# ESS

The (usual) intended use of a (MAP) prior is to reduce the number of control patients in the trial. The prior can be considered equivalent to a number of experimental observations, which is called the effective sample size (ESS) of the prior.

A summary of this explanation based on [@neuenschwander2020predictively] for the estimation of the ESS is presented bellow:

## Moments method

It is based on computing the mean and standard deviation from the parameterized mixture distribution (estimated using EM if it was obtained through simulation). Next, a conjugate distribution that matches these moments allows for easier estimation for the ESS. Which can be derived from the updating rule:

-   **Beta Prior** (for Binomial or Bernoulli data):\
    $$\text{ESS} = a + b$$

-   **Gamma Prior** (for Poisson or Exponential data):\
    $$\text{ESS} = b$$

-   **Normal Prior** (for Normal likelihood with known variance):\
    If the prior is $\mathcal{N}(\mu, \frac{\sigma^2}{n_0})$, then:\
    $$\text{ESS} = n_0$$

Or can be obtained from the variance and precision ratio methods.

### Variance & precision ratio methods

It is based on comparing the variance (or precision) of the prior to the variance of an estimator $Y_N$ for the parameter from a sample of size $N$, averaged over the prior distribution of the parameter. *ESS* is the $N$ for which the two variances are the same.

$$ \text{ESS} = \frac{N \cdot \mathbb{E}_\theta[\text{Var}(Y_N \mid \theta)]}{\text{Var}(\theta)}\text{.} $$

Using $i_F(\theta)$ and $i_F(Y;\theta)$ as the expected and observed Fisher information for one information unit respectively, with:

$$ i_F(\theta) = -\mathbb{E}_{Y_1 \mid \theta} \left[ \frac{d^2}{d\theta^2} \log f(Y_1 \mid \theta) \right]\text{,} $$

we obtain the ESS for the :

$$ \text{ESS}_{\text{VR}} = \frac{\mathbb{E}_\theta[i_F^{-1}(\theta)]}{\text{Var}(\theta)}\text{,} $$

and

$$ \text{ESS}_{\text{PR}} = \frac{1}{\text{Var}(\theta)} \cdot \mathbb{E}_\theta[i_F(\theta)]\text{.} $$

## Morita method

It defines the ESS as the number of observations ($m$) that would make the posterior information from a vague prior with the same mean match the information content of the actual prior.

The ESS is given by:

$$ \text{ESS}_{\text{MTM}} = \frac{i(p(\overline{\theta})) - i(p_0(\overline{\theta}))}{\mathbb{E}_{Y_1}[i_F(Y_1; \theta)]}\text{.} $$

**Where**

-   $i(p(\overline{\theta})) = -\frac{d^2}{d\theta^2} \log p(\theta) \big|_{\theta = \overline{\theta}}\text{ .}$

    → Prior information (curvature of the actual prior).

-   $i(p_0(\overline{\theta})) = -\frac{d^2}{d\theta^2} \log p_0(\theta) \big|_{\theta = \overline{\theta}}\text{ .}$

    → Information of a vague prior with the same mean as $p(\theta)$.

-   $i_F(Y_1; \theta)\text{.}$\
    → Fisher information from one observation.

-   $\mathbb{E}_{Y_1}[\cdot]\text{.}$\
    → Expectation over the prior predictive distribution.

## ELIR method

Instead of evaluating the information ratio at the prior mean or mode, ELIR averages the ratio of prior curvature to Fisher information across all plausible values of the parameter (in this case designed by $\theta$).

$$ \text{ESS}_{\text{ELIR}} = \mathbb{E}_\theta \left[ \frac{i(p(\theta))}{i_F(\theta)} \right]\text{.} $$

**Where**

-   $i(p(\theta)) = -\frac{d^2}{d\theta^2} \log p(\theta)\text{.}$\
    → Prior information (curvature of the log-prior).

-   $i_F(\theta) = \mathbb{E}_{Y \mid \theta} \left[ -\frac{d^2}{d\theta^2} \log f(Y \mid \theta) \right]\text{.}$\
    → Fisher information from one observation.

-   The expectation $\mathbb{E}_\theta[\cdot]$ is taken over the prior distribution.

## Choosing for the method

As previously noted, the lack of convergence in effective sample size (ESS) estimates across different methods necessitates a clear choice of approach. The ELIR method emerges as the only predictively consistent option, making it the preferred technique for accurately quantifying information in terms of an equivalent number of observations. However, its applicability is currently limited to one-dimensional parameters, and further research is needed to address the extension of its benefits to higher-dimensional settings. Additionally, in situations where data-prior conflict is present, the advantages of the ELIR method diminish, given the reduced influence of the prior in the calculation of ESS.

### Beta mixture (RBesT)

The ESS can be calculated in RBesT with the **`ess`** function. The default method in RBesT is the elir approach [@neuenschwander2020predictively] which results in reasonable ESS estimates. The moment matching approach leads to conservative (small) ESS estimates while the Morita [@morita2008determining] method tends to estimates liberal (large) ESS estimates when used with mixtures:

```{r}
# Prior log-density
log_prior <- function(p) log(dlogitnorm_mixture(p, weights, mus, sigmas))

# Second derivative of log-prior
log_prior_prime2 <- function(p) {
  eps <- 1e-5
  (log_prior(p + eps) - 2 * log_prior(p) + log_prior(p - eps)) / eps^2
}

# ELIR integrand
elir_integrand <- function(p) {
  if (any(p <= 0 | p >= 1)) return(0)
  prior_density <- dlogitnorm_mixture(p, weights, mus, sigmas)
  prior_curvature <- -log_prior_prime2(p)
  fisher_info <- 1 / (p * (1 - p))
  ratio <- prior_curvature / fisher_info
  val <- ratio * prior_density
  if (any(is.nan(val)) || any(is.infinite(val))) return(0)
  return(val)
}


# Numerical integration over response scale
elir <- round(integrate(elir_integrand, lower = 1e-5, upper = 1 - 1e-5)$value)
elir
```

### Normal mixture

If we prefer to work with the mixture of normals on the log-odds scale, we must account for the scale parameter of the likelihood, as the Normal-Normal Hierarchical Model (NNHM) is defined with known σ.

As previously mentioned, we approximated the binary endpoint using the function `escalc` to obtain the effect size and corresponding standard deviation for each study. However, the global scale parameter across studies also needs to be approximated based on the known standard deviations. `Bayesmeta` provides two approaches for this: we can obtain the global scale using the function `uisd`, or we can define the scale using a proxy for the binary endpoint that depends on the mean value of theta:

$$ \text{Proxy}(\mu) = 2 \cosh\left(\frac{\mu}{2}\right) $$

-   Using the proxy of the binomial likelihood

# Robustification

In order to avoid type‑I error inflation in the presence of a prior–data conflict (when the results of the new study strongly clash with the historical data), it is recommended to robustify the MAP distribution. RBesT incorporates this step directly into its workflow.

## RBesT

```{r}
map_robust <- robustify(map,
                        weight = 0.2,
                        mean = 1 / 2)
print(map_robust)

# Generate sequence of points over a reasonable domain
x_vals <- seq(0, 1, length.out = 1000)

# Calculate densities
map_dens <- dmix(map, x_vals)
map_robust_dens <- dmix(map_robust, x_vals)

# Plot original MAP
plot(x_vals, map_dens, type = "l", lwd = 2, col = "blue",
     main = "MAP vs Robustified MAP Density (RBesT: beta mixture)",
     xlab = "x", ylab = "Density")

# Overlay robustified MAP
lines(x_vals, map_robust_dens, col = "red", lwd = 2)

# Add legend
legend("topright", legend = c("MAP", "Robust MAP"),
       col = c("blue", "red"), lwd = 2)
```
